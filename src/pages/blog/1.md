<h2 data-locale-en="Deploying the Backend: Beyond the Code" data-locale-es="Desplegar el backend: mas alla del codigo">Deploying the Backend: Beyond the Code</h2>

<p data-locale-en="Ever since I started working on my application's backend, I knew the moment would come when I'd need to put it into production. This stage came with a set of challenges that weren't so much about writing code, but about diving into system administration and understanding how a server works, how to configure it, keep it secure, and ensure the application is always available." data-locale-es="Desde que empece a trabajar en el backend de mi aplicacion, sabia que eventualmente llegaria el momento de ponerlo en produccion. Esta etapa venia cargada de desafios que no tenian tanto que ver con escribir codigo, sino con involucrarme en la administracion de sistemas y comprender como funciona un servidor: como configurarlo, mantenerlo seguro y asegurar que la aplicacion este siempre disponible.">Ever since I started working on my application's backend, I knew the moment would come when I'd need to put it into production. This stage came with a set of challenges that weren't so much about writing code, but about diving into system administration and understanding how a server works, how to configure it, keep it secure, and ensure the application is always available.</p>

<p data-locale-en="In this post, I want to share the journey I took to bring the backend into production. I won't go into sensitive technical details, for security reasons, but I will explain what tasks were necessary, how they impacted the code, and what I learned along the way." data-locale-es="En este post quiero compartir el camino que recorri para llevar el backend a produccion. No voy a entrar en detalles tecnicos sensibles (por razones de seguridad), pero si voy a contar que tareas fueron necesarias, como impactaron en el codigo y que aprendi a lo largo del proceso.">In this post, I want to share the journey I took to bring the backend into production. I won't go into sensitive technical details, for security reasons, but I will explain what tasks were necessary, how they impacted the code, and what I learned along the way.</p>

<h2 data-locale-en="First Steps with the Server" data-locale-es="Primeros Pasos con el Servidor">First Steps with the Server</h2>

<p data-locale-en="After purchasing a VPS service, the first step was connecting to the server remotely via SSH to make sure everything was up to date and to perform basic maintenance tasks. From there, I began preparing the environment needed for the backend to run in a stable and secure way." data-locale-es="Despues de contratar un servicio de VPS, el primer paso fue conectarme remotamente al servidor por SSH para verificar que todo estuviera actualizado y realizar tareas basicas de mantenimiento. A partir de ahi, comence a preparar el entorno necesario para que el backend pudiera ejecutarse de forma estable y segura.">After purchasing a VPS service, the first step was connecting to the server remotely via SSH to make sure everything was up to date and to perform basic maintenance tasks. From there, I began preparing the environment needed for the backend to run in a stable and secure way.</p>

<p data-locale-en="Among other things, I took care of updating the operating system, managing users, permissions, and services, all with the goal of creating an isolated and reliable environment to run the application." data-locale-es="Entre otras cosas, me ocupe de actualizar el sistema operativo, administrar usuarios, permisos y servicios, con el objetivo de crear un entorno aislado y confiable para ejecutar la aplicacion.">Among other things, I took care of updating the operating system, managing users, permissions, and services, all with the goal of creating an isolated and reliable environment to run the application.</p>

<h2 data-locale-en="Choosing and Configuring the Node.js Version" data-locale-es="Elegir y Configurar la Version de Node.js">Choosing and Configuring the Node.js Version</h2>

<p data-locale-en="Since the backend is built with Node.js, I needed a flexible way to install and manage versions. I chose a tool that allowed me to maintain multiple versions on the server and switch between them easily depending on the project's needs." data-locale-es="Como el backend esta desarrollado con Node.js, necesitaba una forma flexible de instalar y gestionar versiones. Me incline por una herramienta que me permitiera mantener diferentes versiones en el servidor y cambiar entre ellas facilmente segun las necesidades del proyecto.">Since the backend is built with Node.js, I needed a flexible way to install and manage versions. I chose a tool that allowed me to maintain multiple versions on the server and switch between them easily depending on the project's needs.</p>

<h2 data-locale-en="Security: Controlling Server Access" data-locale-es="Seguridad: Controlar el Acceso al Servidor">Security: Controlling Server Access</h2>

<p data-locale-en="A key part of the process was establishing basic firewall rules. I enabled the OS firewall and only allowed connections necessary for managing and running the application. This added a protective layer, limiting the system's exposure to unwanted access." data-locale-es="Una parte fundamental del proceso fue establecer reglas basicas de firewall. Active el firewall del sistema operativo y permiti unicamente las conexiones necesarias para la gestion y el funcionamiento de la aplicacion. Esto anade una capa de proteccion, limitando la exposicion del sistema a accesos no deseados.">A key part of the process was establishing basic firewall rules. I enabled the OS firewall and only allowed connections necessary for managing and running the application. This added a protective layer, limiting the system's exposure to unwanted access.</p>

<h2 data-locale-en="Keeping the App Alive" data-locale-es="Mantener la App en Funcionamiento">Keeping the App Alive</h2>

<p data-locale-en="A production app can't afford to stop if a process fails or the server restarts. That's why I incorporated a process manager that runs the backend in the background, restarts it automatically on errors, and keeps it active after a system reboot. There are many options out there for this, such as pm2, forever, systemd, among others." data-locale-es="Una aplicacion en produccion no puede detenerse si falla un proceso o si el servidor se reinicia. Por eso, incorpore un sistema de gestion de procesos que permite ejecutar el backend en segundo plano, reiniciarlo automaticamente ante errores y mantenerlo activo tras un reinicio del sistema. Existen muchas opciones en el mercado para esto, como pm2, forever, systemd, entre otras.">A production app can't afford to stop if a process fails or the server restarts. That's why I incorporated a process manager that runs the backend in the background, restarts it automatically on errors, and keeps it active after a system reboot. There are many options out there for this, such as pm2, forever, systemd, among others.</p>

<h2 data-locale-en="Exposing the Backend to the World" data-locale-es="Exponer el Backend al Mundo">Exposing the Backend to the World</h2>

<p data-locale-en="To allow other services or applications to access my backend, I set up a reverse proxy server to forward HTTP/HTTPS requests to the actual backend service. There are many solutions for this, like Nginx, Apache, or Caddy, each with its own strengths depending on the case. I also added an SSL certificate to ensure that all connections were secure." data-locale-es="Para que otros servicios o aplicaciones pudieran consumir mi backend, configure un servidor intermedio que actua como proxy, redirigiendo las solicitudes HTTP/HTTPS a mi servicio real. Hay muchas soluciones disponibles para esto, como Nginx, Apache o Caddy, cada una con sus ventajas segun el caso. Tambien incorpore un certificado SSL para asegurar que las conexiones fueran seguras.">To allow other services or applications to access my backend, I set up a reverse proxy server to forward HTTP/HTTPS requests to the actual backend service. There are many solutions for this, like Nginx, Apache, or Caddy, each with its own strengths depending on the case. I also added an SSL certificate to ensure that all connections were secure.</p>

<h2 data-locale-en="Code Adjustments for Production" data-locale-es="Ajustes de Codigo para Produccion">Code Adjustments for Production</h2>

<p data-locale-en="Interestingly, some of the challenges related to the production environment led me to modify parts of the code." data-locale-es="Curiosamente, varios de los desafios vinculados al entorno de produccion me llevaron a modificar partes del codigo.">Interestingly, some of the challenges related to the production environment led me to modify parts of the code.</p>

<p data-locale-en="Due to business logic and some limitations of the hosting environment I use, I had to adapt parts of the backend." data-locale-es="Debido a la logica de negocio y a algunas limitaciones del entorno de hosting que utilizo, tuve que adaptar parte del backend.">Due to business logic and some limitations of the hosting environment I use, I had to adapt parts of the backend.</p>

<p data-locale-en="One of the changes involved adjusting the access control system (CORS) to only allow requests from specific domains. This required restructuring how the whitelist was defined and how the backend decided which database to use for each origin." data-locale-es="Uno de los cambios fue ajustar el sistema de control de acceso (CORS) para permitir solicitudes solo desde dominios especificos. Esto implico reestructurar como se definia la whitelist y como se decidia que base de datos usar para cada origen.">One of the changes involved adjusting the access control system (CORS) to only allow requests from specific domains. This required restructuring how the whitelist was defined and how the backend decided which database to use for each origin.</p>

<p data-locale-en="These changes weren't trivial, but they ultimately helped me create a much more flexible and secure structure for the backend." data-locale-es="Estos cambios no fueron triviales, pero terminaron ayudandome a construir una estructura mucho mas flexible y segura para el backend.">These changes weren't trivial, but they ultimately helped me create a much more flexible and secure structure for the backend.</p>

<h2 data-locale-en="Conclusion" data-locale-es="Conclusion">Conclusion</h2>

<p data-locale-en="Deploying an application's backend is a task that goes far beyond simply uploading code to a server. It involves understanding how the operating system works, configuring supporting services, securing the environment, and adapting the code to function properly in that context." data-locale-es="Desplegar el backend de una aplicacion es una tarea que va mucho mas alla de subir codigo a un servidor. Implica entender como funciona el sistema operativo, configurar servicios de apoyo, asegurar el entorno y adaptar el codigo para que funcione correctamente en ese contexto.">Deploying an application's backend is a task that goes far beyond simply uploading code to a server. It involves understanding how the operating system works, configuring supporting services, securing the environment, and adapting the code to function properly in that context.</p>

Today, I can say it was totally worth it.</p>

Es parte del viaje.">If you're going through a similar process, my advice is: take it slow, try to understand what you're doing at each step, and don't be afraid to tweak your code to fit the realities of the production environment.
