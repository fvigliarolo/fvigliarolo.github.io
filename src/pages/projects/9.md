## About this project

ServiPuntosUy is a multi-tenant loyalty platform designed for gas station chains.

The system allows each tenant to manage its own loyalty configuration, promotions, branches, users, and visual identity, while sharing the same technological base. The project includes a .NET backend, an administrative backoffice built with .NET MVC, an end-user web application in React, and a mobile application built with .NET MAUI.

The platform was created as part of a college project, but the scope pushed us to think beyond a basic CRUD application. We had to design a system with different user roles, tenant-aware behavior, QR-based redemption flows, and support for features such as notifications and age verification through a mock external service.

## Walkthrough

One of the most important challenges in this project was the multi-tenant architecture.

Instead of creating isolated deployments for each client, we implemented a shared platform where tenant resolution depends on the request context. For administrators, tenant and role are identified through JWT claims after login. For end users, the system can resolve the tenant from the subdomain, and for mobile scenarios we also supported tenant resolution through headers. This allowed the same backend to serve different experiences depending on who was making the request.

From an architectural perspective, we worked with a layered structure and a service organization that separated behavior by user type. This made it easier to model the responsibilities of platform administrators, tenant administrators, branch administrators, and end users without collapsing all the logic into the same service layer.

Another interesting part of the project was the loyalty flow itself. The platform supports points accumulation, product and promotion management, branch-specific operations, transaction history, and QR-based redemption. That last feature was especially useful as a concrete example of how the different applications connect: the user generates a QR from the app, a branch processes it, and the backend validates and records the redemption.

The project also required us to think about much more than endpoints. We had to work on environment configuration, CORS, JWT authentication, database migrations, documentation for teammates, and development setups that simulated tenant subdomains locally. In practice, a big part of the work was making the system understandable and usable by the whole team.

Because this was a group project, collaboration became part of the technical challenge as well. We documented architecture decisions, usage flows, and setup instructions to reduce friction between the backend, frontend, and mobile parts of the system. Looking back, that documentation was not a secondary task. It was necessary to keep the project moving.

## Conclusion

ServiPuntosUy was one of those projects where the most valuable learning did not come from a single feature, but from having to connect many moving parts into one coherent platform.

Working on a multi-tenant system in .NET, while also coordinating with web and mobile clients, forced us to think more carefully about architecture, boundaries, and maintainability. It was a demanding academic project, but it also felt close to the kind of complexity that appears in real software products.
